In my previous blog post, I described how to make the hardware and software setup for an interactive graffiti wall. In this blog post, I will explain what the frontend code does and how you can make your own similar code!

We have three roles in this architecture.

  • Display - shows the results of all user inputs
  • Smartphone - reports all input to the server
  • Server - proxies all events from smartphones to the display

For this kind of setup, we could spread the logic across all these roles. I chose however to try something different: The smartphones are simply passing on events. The server is also simply passing on events. This means that we can implement all logic on the display end of the code. To listen to the smartphones, simply use

devices.addEventListener(eventType, callback)
