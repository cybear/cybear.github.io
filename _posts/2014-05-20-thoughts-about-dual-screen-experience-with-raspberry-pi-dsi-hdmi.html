Late February this year, RasPi.tv published some information about the upcoming official Raspberry Pi DSI touch screen.

Here's some specs:

* 10 point capacitive touch
* 7 inch LCD
* DSI connector
* 800 x 480 pixels

It sounds like we can expect simultaneous output on both the touch screen and over HDMI.

Given the premise that we have two screens I started imagining how software could make use of it.

• Presentation software - speaker's notes and navigation on touch screen, slides on HDMI
• XBMC - browse, preview, playback controls on touch screen, movie on. HDMI

If the two-screen setup takes off in the Raspberry Pi community, I think we are going to see a lot of software to work with 2 screens.

An idea is to start implementing 2-screen app interfaces in HTML. This way they can be made available to smartphones as well. In many situations I'd love it if I could use a dedicated screen (Pi DSI screen) and my smartphone interchangably.
